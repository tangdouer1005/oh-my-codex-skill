# 保研申请与面试编排

先阅读并按需使用这些已有 skill：
- `lark-shared`：认证、scope、`--as user`
- `lark-mail`：读取申请邮件、发送通知邮件
- `lark-sheets`：维护主表
- `lark-drive`：上传简历
- `lark-doc`：创建空白面试记录文档
- `lark-calendar`：创建/修改飞书会议
- `lark-contact`：把负责人姓名或拼音查成 `open_id`

## 目标对象

默认维护一张飞书电子表格，字段顺序为：

1. `姓名`
2. `简历`
3. `AI介绍`
4. `备注`
5. `邮箱`
6. `面试时间`
7. `负责人`
8. `面试记录`

如果用户没有指定新结构，优先沿用这 8 列，不随意改列顺序。

## 核心约定

- 新候选人默认来自飞书邮箱 `INBOX` 中的转发申请邮件。
- 只把“新的申请信”补进表格，不把候选人回复、接受日程、时间变更确认重复入表。
- 去重优先按 `姓名`，必要时结合学校/邮箱二次确认。
- `邮箱` 默认以简历邮箱为准；如果邮件发件邮箱和简历邮箱不同，优先简历邮箱，必要时在 `备注` 里说明。
- `AI介绍` 优先依据简历生成，不只根据邮件正文。
- 未安排面试的人：`面试时间 / 负责人 / 面试记录` 保持空白。
- 只有已安排面试的人才创建 `面试记录` 空白文档。
- 新创建的 `面试记录` 文档，创建后应立即把公开权限改成：`组织内获得链接的人可编辑（tenant_editable）`。
- 发送邮件前，必须先向用户展示收件人、主题和正文，获得明确同意后再发送。

## 标准工作流

### 1. 从邮箱筛新增申请

- 先读取当前表格，拿到已有姓名列表。
- 用 `lark-cli mail +triage --as user --format json --filter '{"folder":"INBOX"}'` 查看最近邮件。
- 只保留和“保研申请 / 自荐 / 申请 + 学校 + 姓名 / Fwd: ...”相关的邮件。
- 排除候选人回信、会议邀请回复、已接受/暂时接受等通知类邮件。

### 2. 提取候选人信息

对每封新增申请：

- 用 `lark-cli mail +message --as user --html=false --format json --message-id ...` 读取正文和附件。
- 提取：
  - 姓名
  - 学校/专业/排名/GPA/研究方向
  - 联系邮箱
  - 简历附件 ID
- 如果有 PDF 简历，优先下载“简历/个人简历/简历最终版”等主简历文件，不优先下载成绩单或证明材料。

### 3. 下载并上传简历

- 用 `mail user_mailbox.message.attachments download_url` 获取附件下载链接。
- 下载到本地临时目录。
- 用 `lark-cli drive +upload --as user --file './文件名.pdf'` 上传到飞书云空间。
- 在表格第二列填飞书文件链接。

### 4. 生成表格行

新增候选人时，默认写入：

- `姓名`：候选人姓名
- `简历`：上传后的飞书文件链接
- `AI介绍`：1 句简洁总结，突出学校、排名/GPA、主要科研或项目方向
- `备注`：默认可写 `新转发申请。`
- `邮箱`：简历邮箱优先
- `面试时间`：空
- `负责人`：空
- `面试记录`：空

### 5. 安排面试

用户给出排期后：

- 直接更新 `面试时间` 与 `负责人`
- 负责人列优先写真实姓名，不写缩写
- 涉及“周一/周二/明天”等相对日期时，务必换算成绝对日期再写入
- 每次改会议时间时，同时同步表格时间列

### 6. 创建飞书会议

对已安排面试的候选人：

- 用 `lark-contact +search-user` 把负责人查成 `open_id`
- 用 `lark-cli calendar +create --as user` 创建会议
- 标题默认：`面试：姓名`
- 给内部负责人添加参会人
- 给候选人的外部邮箱添加 `third_party` 参会人
- 如需修改时间，用 `calendar events patch`
- 如用户要求删除描述，则保持会议描述为空

### 7. 发送面试通知邮件

默认模板：

```text
{姓名}同学，你好，

这是中国科学技术大学 StarTeam 王皓老师课题组的保研面试通知。你的面试时间安排如下：

时间：{中文日期时间}
会议链接：{meeting_link}

本次面试使用飞书会议进行，请提前下载并登录飞书，确保面试开始前可以正常进入会议。
同时，请提前准备一个 10 分钟左右的 PPT，简要介绍你的个人背景和科研/项目经历。

收到邮件后请直接回复确认。若有任何问题，也可以直接通过邮件与我们联系。

祝好，
唐嘉辉
中国科学技术大学 StarTeam
```

执行顺序：

1. 先整理所有候选人的发送预览给用户确认
2. 用户明确同意后，再用 `lark-cli mail +send --confirm-send`
3. 发送后用 `send_status` 查询投递状态

如果只是更新已通知候选人的面试时间，默认使用这版“时间更新通知”模板：

```text
{姓名}同学，你好，

这是中国科学技术大学 StarTeam 王皓老师课题组的保研面试时间更新通知。你的面试时间现调整如下：

时间：{中文日期时间}
会议链接：{meeting_link}

本次面试仍使用飞书会议进行，请提前下载并登录飞书，确保面试开始前可以正常进入会议。
同时，请提前准备一个 10 分钟左右的 PPT，简要介绍你的个人背景和科研/项目经历。

收到邮件后请直接回复确认。若有任何问题，也可以直接通过邮件与我们联系。

祝好，
唐嘉辉
中国科学技术大学 StarTeam
```

如果用户要求发送未录取通知，默认使用这版“拒信”模板：

```text
{姓名}同学，你好，

感谢你参加我们课题组的面试。你的整体表现很不错，也让我们看到了你的能力和潜力。

但因为今年申请人数很多、竞争非常激烈，同时课题组名额比较有限，经过综合讨论后，我们这次很遗憾暂时无法向你发出录取邀请。

这并不影响我们对你能力的认可，也希望你继续关注其他老师和课题组的机会。相信你后面一定能找到非常合适的方向和导师。

感谢你的时间和投入，祝你后续申请顺利。

唐嘉辉
中国科学技术大学 StarTeam
```

### 8. 面试记录文档

当用户要求补 `面试记录` 列时：

- 为所有“已安排面试”的候选人创建空白 docx 文档
- 标题默认：`面试记录 - 姓名`
- 创建后立即将文档公开权限改为：`组织内获得链接的人可编辑（tenant_editable）`
- 把文档链接写入第 8 列
- 未安排面试的人，该列保持空白

## 输出风格

- 表格更新后，优先告诉用户：
  - 新增/更新了哪些候选人
  - 影响了哪些列
  - 表格链接
- 如果只是小改动，直接说明改了哪一行/哪一场会议，不要长篇复述全表。

## 常见坑

- `docs +create` 需要 `docx:document:create`
- `mail +triage` 需要邮件正文/地址/主题相关 scope
- `drive +upload` 只接受当前目录下的相对路径
- `sheets +append` 的 `--range` 必须能容纳待追加的行数
- 修改会议时间后，会议链接通常不变，但仍应读回确认
- 更新表格时，不要覆盖用户后来手动改过的无关列
